Quirks of the TLB

TLB remembers the mappings between the virtual page number and physical page numbers.
But what happens when we context switch? One way is we need to flush the entire TLB each time.
We're always working for ways to preserve the TLB. All architectures has us using two obvious ways we can preserve the TLB. We can't directly access it via software nor how big it is.

One obvious thing we can do is segregate the part of our virtual memory space. Parts which are private to our process, and parts which are shared by many processes. Shared things stay in TLB.

The ugly part is what happens when we have multiple cores.
Each core gets a TLB

If one core is the only one using a given part on the TLB, we have to convince the other TLBs to flush the shared object.
Anything shared when flushed from one TLB should find a way to flush from other TLBs.
