Notes

Our software had a virtual address, hardware then converted this to a physical address, and that was what got fed to memory.
The translation is not done at the byte level, but by chunking into pages.

How we utilize the virtual address space

The system memory space (which is the upper half of the virtual memory address space) is shared. We're making each thread have it's own tree. That way all data within that thread is private, but we can make system information access shared.
We initialize a new tree which every time has an identical upper half as the shared memory space.
