1.16 Virtual Memory

Overview

Introduced User Mode access, and defined virtual memory. Virtual memory is a user-process specific abstraction so that user programs can have the illusion of contiguous blocks.

Notes

We want to now work on our user level abstraction, where the user works in a confined workspace. Then we put user threads inside sandboxes, so they can't leave the confines of their box. User processes are these things.
Now we want a place for the user processes to come from, which is in the filesystem! Someone says to take a program, load it in memory, then run it. That's a loader

The other option is to tell the person who wrote the program to produce code independent of global position.
