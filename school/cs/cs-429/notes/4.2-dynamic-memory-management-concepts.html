4.2 Dynamic Memory Management Concepts

Overview

We discussed the basics of memory management and memory allocation.

Notes

Memory Management Concepts

Dynamic memory management refers to the algorithms and structures for allocating and deallocating objects of variable size in memory at program execution time.
The sequence data is given to the user, may not be the sequence it is returned to the computer.

  • Lets say we fill in chairs row by row, but some peopled decide to leave early leaving holes! What that means is that we have to find a smart way to keep track of the data filling into the holes
  • We also need an interface for the OS incase the user requests for more memory than you currently have permission for.
    Some examples of the above concepts in C are malloc()/free and new/delete in C++
    The thing that manages this is called the ALLOCATOR. We're designing an explicit memory allocator that the user is responsible for using. In Python and Java, they use an implicit memory allocator (called a garbage collector).
  • We'll learn more about this in OS

The Bump Allocator

You can just keep requesting more memory from the operating system and never free memory up. It's great with time, horrible with space.
