6.1 Pointers

Overview

Today we're learning seven facts about pointers

Notes

Basic Info

To declare the type of a pointer, you use the form T* x;, where T is the target type.

When would you use a pointer to a pointer? Ex char** cpp

You might have a set of pointers to objects, that you then want to switch across. The pointers to the objects themselves don't move, but our outer pointer switches across them!

At the machine level, the pointers themselves memory addresses don't contain any type information! It takes 64 bits to take me to any spot in my memory, so the width of the pointer itself has the same size. That then begs the question, where is the type stored?

The value of a pointer is then the address.
The NULL pointer denotes a num pointer constant which can't be dereferenced.

When a pointer is uninitialized it will hold the address of an arbitrary memory location.

Creating a pointer

If I have an uninitialized pointer int *xp and I want to make it so that I can then initialize it with the location of int x, then I can use the dereference operator &! With int* xp =&x I then make the pointer directed toward where x is located.

You can get the address in multiple ways.
One thing that doesn't make any sense is int **z = &(x&). The address of x is not an L value, it hasn't been stored anywhere so it has no address to dereference to.

The type of a pointer tells us how the pointers should increment when we add to them! An int *p+1 will increment by 8 bytes, while if I casted to (char)*p+1 I'd move forward by 4 characters.

Where is the type of a pointer then stored? Where are types stored at all in C?

Arrays and Casting Pointers

You can reference the name of an array as a pointer variable. The type of an array is just a pointer to the type it targets. *s = 'h's[0]='h'

The width that you jump is equal to the type of your array

Casting a pointer into a different type just changes the interpretation of the existing data. Nothing about the data itself changed.
Casting obeys order of operations, so it slides in the same section as multiplication.
