There are 3 types of char. There are char signed char, unsighed char. These are collectively the character types.

  • In the int worlds, we consider signed ints and regular ints, they're the same type. This isn't so with the characters where they're al different types.

What ranges will char use? You change the setting in the <limits.h>. 

1
CHAR_MIN = -128 -> char is signed
2
CHAR_MIN = 0 -> char is unsigned

Char's will always be the same size as 1 byte. There is no concept of endianness with chars.

ASCII Encoding


ASCII is just a codebook to map existing representations to values. It's an encoding/an interpretation.
Because char is 1 byte long, we would think we'd have 256 representations. Sucks to suck, it's a 7 bit code

ASCII is a 7 bit code
Diagram of any char's type in memory

The set of printable characters had in addition had some on printable character. There were also non-printable characters though!

  • Things like shift, caps-lock, arrows, escape, etc. Those are control characters (0x00-0x1F,0x7F)
  • The rest are printable characters (`0x20-0x7E)
    This range is the code-space! Each pattern is a code point.
NUL has an ASCII code (this is the all zeroes charecter \0), but note that it's not NULL! That is a value

In the ASCII codebook, they do expect to have some of the fields memorized

  • The first 32 are non-printable characters
  • Printables start from hex 20
  • Contiguous characters go from 0x30-0x39
  • Then capital letters is 0x41-0x5A. The difference between upper and lower case is 0x20
How do I go from the character 3 to the value 3?

Well you can just subtract off the character 3 from the character zero! Then your bitwise representation is the same. You would need to cast though so that the interpretation is correct

In C, you have a function atoi() which is a built in ASCII to integer function in <stdllib.h>.

  • Before getting to atoi(), make sure you sanitize inputs first. It will blindly cast into integer representation assuming base 10

We now have a variable length representation for our codespace! ASCII has a fixed length, but now we're allowing for dynamic sizing. The need for a larger code literally exploded.

UTF-8

This is a variable length character encoding. We need to always stay backward compatible, so ASCII must still remain.

  • We can use 1,2,3, or 4 bytes
    When using 1 byte, it's the old ASCII code. Depending on what your needs are, it will dynamically resize.
