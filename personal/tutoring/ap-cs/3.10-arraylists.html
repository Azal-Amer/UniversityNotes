Lesson Plan

Learning Objectives

By the end of this lesson, students will be able to:

  • Explain the key differences between Arrays and ArrayLists
  • Implement basic ArrayList operations (add, remove, get, set)
  • Convert existing Array-based code to use ArrayLists
  • Select appropriate data structure (Array vs ArrayList) for different scenarios

Lesson Timeline

1. Introduction (10 minutes)

Opening Problem

Present this scenario to demonstrate array limitations:

// Ask class: How would we handle a team that grows?
String[] team = new String[3];
team[0] = "Humza";
team[1] = "Abdullah";
team[2] = "Yusuf";
// What happens if a new student joins?

Discussion points:

  • What are the limitations of fixed-size arrays?
  • Real-world scenarios where size changes
  • Cost of resizing arrays manually

2. ArrayList Basics (15 minutes)

Live Coding Demo

// Old way (Arrays)
String[] team = new String[3];

// New way (ArrayList)
ArrayList<String> team = new ArrayList<>();
team.add("Humza");  // No size limit!

Key Concepts to Cover:

  1. Import statement: import java.util.ArrayList;
  2. Basic operations:
    • add(element)
    • remove(index) vs remove(object)
    • get(index)
    • set(index, element)
    • size()
  3. Dynamic sizing
  4. Wrapper classes (Integer vs int)

3. Guided Practice (20 minutes)

  • work through Part 1 and 2 of worksheet
  • help with syntax issues
  • After 10 minutes: Class review of common solutions
  • Group discussion of different approaches

4. Practical Application (10 minutes)

Converting Basketball Assignment

// From previous homework
private int[] scoresByGame;
private boolean[] gameResults;

// To ArrayList version
private ArrayList<Integer> scoresByGame;
private ArrayList<Boolean> gameResults;

// Demonstrate conversion of methods
public void addGameScore(int score) {
    scoresByGame.add(score);
}

5. Wrap-up (5 minutes)

  • Review key differences between Array and ArrayList
  • When to use each type
  • Preview homework assignment
  • Final Q&A

Common Issues to Address

  1. Forgetting import statement
  2. Trying to use primitive types (int, boolean) instead of wrapper classes
  3. Confusing array index notation with ArrayList methods
  4. Understanding dynamic sizing implications
